八、docker镜像的构建
docker镜像的构建有以下两种方式：
1）docker commit  :  基于容器做变更，再将容器打包成新的镜像；
2）Dockerfile  : 基于基础镜像做变更，再使用docker  build命令生成新的镜像。docker  build   -t   $image_name  . 
1、docker  commit手动构建镜像
  • 案例：某容器云厂商的centos镜像闭环管理项目
  • 环境准备：
              配置/etc/hosts :   192.168.100.10    docker
1）将tar包加载为镜像：docker  load -i  centos_latest.tar.gz
2）创建私有仓库：
     docker  run  -d -p 5000:5000 --name registry --restart=always  registry将镜像
     vim   /etc/daemon.json   -> 
     {
            "insecure-registries":["docker:5000"],
            "registry-mirrors":["https://8djob8sk.mirror.aliyuncs.com"]
     }
     systemctl  daemon-reload
     systemctl  restart  docker
3）打上仓库标签：docker  tag $image_ID docker:5000/centos:latest
4）将镜像推送到私有仓库：docker push  docker:5000/centos:latest
      registry容器内部存放镜像的路径：
    docker exec  -it  registry /bin/sh  ->
    /var/lib/registry/docker/registry/v2/repositories
    curl  http://docker:5000/v2/_catalog
    浏览器访问地址：http://192.168.100.10:5000/v2/_catalog
5）将镜像下载至本地：docker pull docker:5000/centos:latest
6）将镜像跑成容器：docker run -itd  docker:5000/centos:latest  /bin/bash
7）进入容器对其做变更：docker  exec  -it  $container_ID  /bin/bash  
             ->  安装一个vim工具
8）将容器制作成镜像：docker  commit  -a "chenglu" -m "centos+vim"  $container_ID  centos:vim  ->  实质是给容器当前状态打一个快照，保存到docker里面。
9）将新的镜像打上仓库标签：docker  tag  $image_ID docker:5000/centos:vim
10）将新的镜像推送到仓库：docker  push  docker:5000/centos:vim
11）将镜像下载到docker远程主机：docker  pull  docker:5000/centos:vim
12）在远程主机上将新的镜像跑成容器：docker  run  -itd  docker:5000/centos:vim /bin/bash
13）到容器里面验证vim是否已安装可用
14）将容器导出为tar包：docker  export  -o  centos_vim.tar  $container_ID  等于 docker  export  $container_ID > centos_vim.tar
15）将镜像导出为tar包：docker save -o centos_vim.tar  $image_ID   等于  docker  save  $image_ID  >  centos_vim.tar
16）查看远程主机的tar包是否生成: ls  ./
2、dockerfile自动化构建镜像
  • 格式：
文件关键字：
    1)FROM :  指定基础镜像（不可缺省）
这里的 基础镜像一定是本地存在的或者官方仓库上已有的，否则它不会被运行成功
    2)MAINTAINER：说明本镜像的维护者信息，包括姓名和邮箱地址（可以缺省），这一层会被自动销毁，因为不重要！
    3)ENV：传递环境变量到镜像中（可以缺省），这一层也不重要，生成过程中也会被销毁。
    4)ADD：复制文件（包括本地文件或者URL链接中的文件或者tar包）到镜像的目标路径中，如果目标路径是一个文件，则该复制覆盖目标路径文件中的内容，其中本地文件必须与Dockerfile在同一个文件夹里面，另外，在构建镜像的时候，tar包会自行解压，后被删除。
    5)COPY：复制文件（仅文件）到镜像的目标路径中，如果目标路径是一个文件，则该复制覆盖该文件的内容，另外，如果目标路径不存在，那么构建镜像的时候，会自动生成该路径。
示例：
vim   Dockerfile  ->
FROM   centos:7
MAINTAINER Cheng lu<182053702@qq.com>
ENV    HOSTNAME "centos7-http"
ADD    CentOS-Base.repo  /etc/yum.repos.d/CentOS-Base.repo
COPY   aaa.txt  /opt/aaa/aaa.txt
docker  build  -t  centos:repo_v1.0  .
docker  run  -itd  centos:repo_v1.0  
docker  exec -it  $container_ID  /bin/bash
-> cd  /opt/aaa  ->  cat  aaa.txt
    6)RUN：指定镜像要做哪些任务
                 格式可以有以下两种，二选一即可：
        a. exec 格式：以中括号括起来，命令的每个部分用引号引起来，命令每个部分之间用逗号隔开。
        b.shell命令格式：即Linux命令
示例：
FROM   centos:7
MAINTAINER Cheng lu<182053702@qq.com>
ENV    HOSTNAME "centos7-http"
ADD    CentOS-Base.repo  /etc/yum.repos.d/CentOS.Base.repo
COPY   index.html  /usr/local/apache2/htdocs/index.html
COPY   aaa.txt  /opt/aaa/aaa.txt
RUN    yum -y install httpd
    7)USER：为后续的指令指定运行的用户身份，如root用户或某个普通用户。
    8)WORKDIR：为后续的指令指定工作路径，一个Dockerfile文件可以指定多个WORKDIR路径。
示例：
FROM   centos:7
MAINTAINER Cheng lu<182053702@qq.com>
ENV    HOSTNAME "centos7-http"
ADD    CentOS-Base.repo  /etc/yum.repos.d/CentOS.Base.repo
COPY   aaa.txt  /opt/aaa/aaa.txt
#RUN    yum -y install httpd
RUN    rm -f /opt/aaa/aaa.txt
WORKDIR  /root/centos
RUN   echo "hello centos" >> centos.txt
    9）VOLUME：docker run -v /var/lib/docker/${文件系统名称}目录下 :{文件系统名称}目录下
示例：
FROM   centos:7
MAINTAINER Cheng lu<182053702@qq.com>
ENV    HOSTNAME "centos7-http"
ADD    CentOS-Base.repo  /etc/yum.repos.d/CentOS.Base.repo
COPY   aaa.txt  /opt/aaa/aaa.txt
#RUN    yum -y install httpd
VOLUME ["/data"]
RUN    rm -f /opt/aaa/aaa.txt
WORKDIR  /root/centos
RUN   echo "hello centos" >> centos.txt
    10）EXPOSE  ：容器要暴露的端口号
示例：
FROM   centos:7
MAINTAINER Cheng lu<182053702@qq.com>
ENV    HOSTNAME "centos7-http"
ADD    CentOS-Base.repo  /etc/yum.repos.d/CentOS.Base.repo
COPY   aaa.txt  /opt/aaa/aaa.txt
#RUN    yum -y install httpd
VOLUME ["/data"]
RUN    rm -f /opt/aaa/aaa.txt
WORKDIR  /root/centos
RUN   echo "hello centos" >> centos.txt
EXPOSE 80
    11）HEALTHCHECK：健康检查，检查容器进程是否出现故障卡死导致不能退出，若是，则强制退出容器。
    12）CMD ： 指定容器启动的第一条命令是什么（容器启动后要做的第一个任务），注意，该指令指定的任务会被docker run 指定的任务覆盖。
示例：
FROM centos:7
RUN yum -y install httpd
VOLUME ["/var/www/html"]
CMD ["httpd","-D","FOREGROUND"]  #或者CMD /usr/sbin/apachectl -D FOREGROUND
WORKDIR /var/www/html
EXPOSE 80
    13）ENTRYPOINT：指定容器启动的第一条命令是什么（容器启动后要做的第一个任务），注意，该指令指定的任务不会被docker run 指定的任务覆盖。
示例一：在entrypoint关键字中，使用shell脚本实现开启新终端的操作
[root@centos dockerfile]# cat Dockerfile
FROM ubuntu
COPY entrypoint.sh /entrypoint.sh
RUN chmod +x /entrypoint.sh
ENTRYPOINT ["/entrypoint.sh"]
[root@centos dockerfile]# cat entrypoint.sh
#!/bin/bash
exec bash
示例二：在entrypoint关键字中，使用shell脚本实现开启HTTP的操作，并挂载主页文件到镜像中
[root@centos dockerfile]# cat Dockerfile
FROM centos:7
RUN yum -y install httpd
COPY entrypoint.sh /entrypoint.sh
VOLUME ["/var/www/html"]  #通过本地路径“/var/lib/docker/${文件系统名称}”传递主页文件到/var/www/html中
RUN chmod +x /entrypoint.sh
ENTRYPOINT ["/entrypoint.sh"]
WORKDIR /var/www/html
EXPOSE 80
[root@centos dockerfile]# cat entrypoint.sh
#!/bin/bash
httpd -D FOREGROUND
